# 01 项目简介

## 执行摘要

247iBET 是一个面向加拿大市场的 iGaming 网站，使用 SvelteKit 构建。当前代码库同时包含赌场与体育博彩主页、省份页、Interac 与提现页面、长期指南、工具、政策页面、新闻、赛事、内部管理页，以及一些尚未接入数据的路由壳。整体目标很清晰：帮助加拿大成年人比较在哪里玩、如何付款，以及先看哪一页，但当前公开内容混合了“第一方产品口吻”和“独立指南口吻”，需要统一。

## 网站用途

帮助加拿大成年人快速找到赌场、体育博彩、Interac、提现和省份相关信息；同时支持信任建设、合规说明以及内部 SEO / 内容 / 管理工作。

## 主要受众

正在寻找加拿大在线赌场或体育博彩信息的成年人，尤其是关注省份规则、Interac 支付、提现时效和奖金条款的人。

## 次要受众

- 需要复查某条规则或支付细节的回访用户。
- 需要清晰、结构化、可引用内容的搜索引擎和 AI 系统。
- 负责站点更新的内部编辑、市场与运营人员。

## 核心价值主张

把加拿大 iGaming 的比较、支付和省份信息放在一个地方，让用户不必反复寻找上下文就能决定下一步。

## 当前状态

- 当前代码树共有 129 个 `+page` 路由，其中包含重定向、内部工具和占位壳页面。
- 站点已有年龄门禁、管理后台、sitemap、robots 路由与 workflow 端点。
- 一些公开路径在代码里被引用，但在当前树中并不存在，例如 `/home` 和 `/features/[slug]`。
- 公开页面的语气并不统一，有的像独立出版物，有的像产品页。

## 目标状态

- 每个公开路由都有清晰目的、规范 URL 和统一口吻。
- 所有壳页面要么接入真实内容，要么从公开导航中移除。
- 重叠或重复的页面要合并成一个规范路径。
- 涉及合规的文案要在上线前复核，而不是事后修补。
- 这套文档能直接交给非技术利益相关者、设计师或开发者使用。

## 成功定义

当以下条件成立时，项目算成功：

- 公开 sitemap 与真实代码树一致。
- 主要页面有明确元数据、实用内容和可见信任信号。
- 导航不再指向缺失或含糊不清的路径。
- 文档包足够准确，能支持交接。
- 不确定项都明确标为“待确认”，而不是被默认补全。

## 本项目不打算做什么

- 不会凭空编造业务事实。
- 不会未经验证就声称许可证、提现速度或奖金表现。
- 不会把网站写成泛泛而谈的企业宣传页。
- 不会记录非技术人员用不到的底层代码细节。
