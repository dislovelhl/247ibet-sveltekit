## 元数据要求

- 每个可索引的公开页面都必须有唯一标题。
- 每个可索引页面都必须有简洁且有用的 meta description。
- canonical 必须指向唯一正确的版本。
- 高意图页面应包含可见的 OG 元数据和结构化数据。
- 不打算参与排名的页面应设置为 `noindex` 或从可索引发现中排除。

## 内链计划

- 首页应链接到主中心页和信任页。
- 中心页应链接到最有价值的分支页。
- 省份页应链接到法律、支付与支持页。
- 奖金页应链接到 wagering 解释和条款页。
- 工具页应链接到相关指南和中心页。
- 信任页应链接到联系、责任博彩和来源页。

## 结构化数据建议

- 通用着陆页使用 `WebPage`。
- 永续指南、新闻和说明类页面使用 `Article`。
- 页面上真的可见问答时再使用 `FAQPage`。
- 有清晰层级关系的页面加 `BreadcrumbList`。
- 联系页使用 `ContactPage`。
- 只有当可见内容支持时，才使用 `Organization` / `NewsArticle`。

## 本地 / 地理 SEO 建议

- 清楚区分安大略与阿尔伯塔页面。
- 让省份法律与年龄说明可见。
- 省份中心页要回链到法律、支付和支持页。
- 不要把省份规则混进通用全国页面而不说明差异。

## 技术 SEO 清单

- Canonical 能正确解析。
- 重定向是有意为之且已记录。
- `robots.txt` 与真实路由树一致。
- `sitemap.xml` 只包含应被抓取的页面。
- 搜索页保持 `noindex, follow`。
- 图片 alt 文本有意义。
- 标题层级正确且唯一。
- JSON-LD 与可见内容一致。

## 内容刷新计划

- 按固定周期复审高风险内容。
- 先刷新法律、年龄、提现、奖金和省份页面。
- 市场变化时立即重查时效敏感页面。
- 在占位壳页面变旧之前先更新或移除它们。

## 测量计划

- 用 Google Search Console 看曝光、点击和查询覆盖。
- 用分析工具或服务器日志看页面参与和点击路径。
- 用人工审核检查元数据、canonical 和结构化数据。
- 用内容审计记录追踪时效敏感或合规敏感页面。
