# 06 内容策略

## 信息主轴

1. **加拿大清晰性** —— 用通俗语言解释省份规则、可用性和账户预期。
2. **支付信心** —— 让用户一眼看懂 Interac、存款、提现和时效。
3. **决策支持** —— 帮用户做比较，而不是硬推购买。
4. **信任与安全** —— 把披露、验证、安全和责任博彩放在显眼位置。
5. **以实用为先** —— 每页都应该回答真实问题，或帮助完成真实任务。

## 语气

- 使用通俗、直接、中文读者和英文读者都容易理解的表达。
- 直接、冷静。
- 自信，但不夸张。
- 对监管类表述要谨慎。
- 每类页面只保留一种统一语气。

### 语气提醒

当前代码树里，第一方产品口吻和独立指南口吻混在一起。这不是小问题，而是定位问题。公开站点应针对不同页面族群采用统一口吻，而不是把两种口吻混着写。

## 首页内容要求

- 清楚说明这个站点能帮用户做什么。
- 让用户能快速进入 casino、sportsbook、Interac、省份和指南内容。
- 展示可见的信任与披露信息。
- 首页推荐内容必须是真正上线的规范页面。
- 最好有一小段 FAQ 或帮助说明。
- 主要 CTA 附近要放责任博彩链接。

## 着陆页内容要求

每个高意图着陆页都应该包含：

- 开头就给出直接答案。
- Hero 里明确说明页面主题。
- 一个比较或解释模块。
- 一个信任或披露模块。
- 一个清晰 CTA。
- 相关内链。
- 规范 URL 和页面级元数据。

## 博客 / 指南 / 内容中心要求

- 先给答案，再展开解释。
- 用符合用户提问方式的副标题。
- 需要比较时用表格。
- 把取舍写清楚。
- 加上最后审阅或发布日期。
- 链回中心页和最相关的支持页。
- 只有当 FAQ 真正在页面上可见时，才加入 FAQ。

## 信任内容

优先级最高的信任页：

- `about/how-we-test`
- `about/affiliate-disclosure`
- `about/brand-identity`
- `editorial-policy`
- `security`
- `sources`
- `responsible-gambling`
- `contact`

## FAQ 策略

- 只在问题确实有帮助的高意图页面上使用 FAQ。
- 如果页面上没有可见问答，不要加 FAQ schema。
- 问题要短，答案要具体。
- FAQ 是用来降低不确定性，而不是凑字数。

## 合规安全措辞提醒

- 未经验证，不要写成“保证提现”、“保证奖金”或“保证许可”。
- 如果页面本来是信息页，不要把奖金文案写成直接诱导。
- 省份法律引用要谨慎，并保持当前。
- 对尚未核实的内容使用“待确认”。
- 任何可能快速变化的声明，都应该当作复审项，而不是固定事实。

## 内容缺口

- 多个动态路由壳页面仍然提示需要 CMS 接口。
- 一些页面仍包含占位比较内容或 TODO 名单。
- 评测/功能路径需要一个唯一规范路由。
- 公开语气需要统一，不能让读者猜测这到底是编辑内容还是产品内容。

## 内容生产优先级

1. 先修复缺失 / 错误的规范路径与路由壳。
2. 完成首页、casino、sportsbook、Interac 和省份页。
3. 补齐或移除占位的新闻、赛事、支付、评测与指南壳页面。
4. 更新信任页与来源页。
5. 核心页面稳定后，再扩展指南和工具。
