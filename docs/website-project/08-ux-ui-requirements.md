# 08 UX / UI 要求

## 设计原则

- 首屏就能看懂页面意图。
- 视觉上要高级、安静，不要做成嘈杂的促销页。
- 主要动作要清晰，次要动作仍然可见。
- 在用户做决定的位置放置信任与合规信号。
- 不要用布局技巧隐藏重要信息。

## 布局要求

- 重点着陆页需要强 Hero。
- 桌面端要保持可读的内容宽度。
- 用卡片、表格和小标题拆分长内容。
- 深层页面要加面包屑。
- 相关链接应放在正文后部，方便自然继续浏览。

## 移动端要求

- 主要动作要适合拇指操作。
- 要考虑安全区（safe-area）
- 除非表格确实需要，否则避免横向滚动。
- Sticky CTA 要可见，但不能挡住关键内容。
- 所有可点击区域都要足够大。

## 无障碍要求

- Skip link 必须可用。
- 标题层级要正确。
- 焦点态要明显。
- 颜色对比度要足够。
- 表单控件要有清楚标签。
- 真正的操作必须用真实按钮。
- FAQ / 手风琴等组件要支持键盘操作。

## CTA 要求

- 每个区块只保留一个主 CTA。
- 按钮文字要直白。
- 不要让每个链接都像促销按钮。
- 外部合作链接要有安全属性和清晰意图。
- CTA 文案要和页面目的匹配。

## 信任信号位置

- 高意图页面的 Hero 附近。
- 比较表附近。
- 奖金区块附近。
- 页脚附近。
- 用户即将行动的页面上，而不是只放在政策页上。

## 表单要求

- 每个字段都要有标签。
- 解释为什么需要这个字段。
- 校验错误要贴近字段显示。
- 出错后要保留已输入内容。
- 成功与失败状态都要清楚可见。
- 不要把必填字段藏在图标或占位符里。

## 导航要求

- 主导航要常驻。
- 相关链接要分组。
- 不要把多个不同名字都指向同一个目的地。
- 页脚应是补充层，不是主导航的重复拷贝。

## 错误 / 空状态 / 加载状态要求

- 空状态要说明缺什么，以及下一步怎么做。
- 加载状态不能像坏掉了一样。
- 占位页必须明确告诉用户它还没完成。
- 重定向要看起来是有意设计，而不是错误。
- 缺失内容不能伪装成已完成内容。

## 视觉层级建议

- 先看 Hero 标题。
- 然后是简短的说明段落。
- 再是信任信号或摘要。
- 用户拿到足够上下文后，再给主 CTA。
- 次级链接放在正文后部，不要抢主视线。
