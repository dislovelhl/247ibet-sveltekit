# AGCO/iGaming Ontario Completion Audit — 2026-05-12

## Objective restated

Make the 247iBET SvelteKit repository AGCO/iGaming Ontario safe for a public brand, SEO, acquisition, affiliate/operator-routing, responsible-gambling, and integration-shell launch, while respecting that the real gaming platform is separate.

The user states that a licence exists. This repository still requires documentary proof before public pages may claim that 247iBET is AGCO licensed, iGaming Ontario registered, or an Ontario regulated operator.

## Success criteria

1. Public copy must not claim 247iBET is AGCO licensed, iGaming Ontario registered, fully regulated, or otherwise regulator-approved unless documentary evidence is present.
2. The repo must preserve the architecture boundary: public web shell here; gaming/account/wallet/KYC/payment/odds/game execution in the separate gaming platform.
3. Public pages must not pretend to execute gaming functions, process bets, complete deposits/withdrawals, own a wallet, or provide account-level RG controls.
4. Affiliate handoff CTAs must be marked and disclosed.
5. Responsible-gambling content must use current official-source framing and safe age/province language.
6. Bonus and promotion copy must avoid public inducement and exact current-offer values unless approved by compliance.
7. Future licence enablement must be documented as an evidence-gated change.
8. Verification must cover source copy, brand config, docs, and build health.

## Supplemental sweep — public operational-claim cleanup

After the first audit, a broader source scan found additional public-copy risks outside the initial high-intent route set:

- exact or near-exact payout timing claims in About, Fast Payouts, Interac, CAD casino, and ranking pages;
